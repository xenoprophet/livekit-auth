# LiveKit Auth Service

Minimal token service for LiveKit. Accepts a Mumble username and channel ID, returns a signed LiveKit JWT. That's it.

## Files

```
livekit-auth-service/
├── main.py            # FastAPI app
├── requirements.txt   # Python deps
├── Dockerfile
├── docker-compose.yml
├── .env.example
└── README.md
```

## Setup

1. Copy `.env.example` to `.env` and fill in your LiveKit credentials:

```bash
cp .env.example .env
```

Your LiveKit API key, secret, and URL come from your `livekit.yaml` config:

```yaml
# livekit.yaml (on your LiveKit server)
port: 7880
keys:
  your_api_key: your_api_secret
```

Set `LK_URL` to your LiveKit server's WebSocket address (e.g. `ws://192.168.1.50:7880`).

2. Build and run:

```bash
docker compose up -d
```

## Usage

### Request a token

```bash
curl -X POST http://localhost:3000/token \
  -H "Content-Type: application/json" \
  -d '{"identity": "username", "room": "channel-id"}'
```

### Response

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "url": "ws://192.168.1.50:7880"
}
```

### Health check

```bash
curl http://localhost:3000/health
```

### API docs (auto-generated by FastAPI)

```
http://localhost:3000/docs
```

## Electron Client Usage

```js
async function getLiveKitAuth(username, channelId) {
  const res = await fetch('http://localhost:3000/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ identity: username, room: channelId })
  })
  const { token, url } = await res.json()
  return { token, url }
}

// On Mumble channel join
mumbleClient.on('channelJoin', async (channel) => {
  const { token, url } = await getLiveKitAuth(username, channel.id)
  await livekitRoom.connect(url, token)
})

// On Mumble channel leave
mumbleClient.on('channelLeave', () => {
  livekitRoom.disconnect()
})
```

## GStreamer Setup (Required for Video/Screenshare)

GStreamer is used for GPU-accelerated video encoding. Install it on each client machine — the Electron app detects it at runtime. Video/screenshare is disabled if GStreamer is not found; voice and chat work regardless.

### Windows

1. Download the **runtime installer** (MSVC 64-bit) from https://gstreamer.freedesktop.org/download/
2. Run the MSI installer — use the **Complete** install to include all plugins
3. Ensure GStreamer is added to your PATH (the installer offers this option)
4. Verify: open a terminal and run `gst-launch-1.0 --version`

### Linux

**Debian/Ubuntu:**
```bash
sudo apt install gstreamer1.0-tools gstreamer1.0-plugins-base gstreamer1.0-plugins-good gstreamer1.0-plugins-bad gstreamer1.0-plugins-ugly gstreamer1.0-vaapi
```

**Fedora:**
```bash
sudo dnf install gstreamer1 gstreamer1-plugins-base gstreamer1-plugins-good gstreamer1-plugins-bad-free gstreamer1-plugins-ugly-free gstreamer1-vaapi
```

**Arch:**
```bash
sudo pacman -S gstreamer gst-plugins-base gst-plugins-good gst-plugins-bad gst-plugins-ugly gst-plugin-va
```

### macOS

**Homebrew:**
```bash
brew install gstreamer
```

Or download the .pkg installer from https://gstreamer.freedesktop.org/download/

### Verify Installation

On any platform, run:
```bash
gst-launch-1.0 --version
```

You should see output like `GStreamer 1.24.x` or similar.

### GPU Encoder Support

| Platform | GPU | Encoder Plugin | Driver Required |
|----------|-----|----------------|-----------------|
| Windows/Linux | NVIDIA | `nvh264enc` | NVIDIA driver 470+ |
| Windows | AMD | `amfh264enc` | Adrenalin driver |
| Linux | AMD/Intel | `vaapih264enc` | Mesa VA-API |
| macOS | Apple Silicon/Intel | `vtenc_h264` | Built-in (no extra driver) |

If no GPU encoder is available, GStreamer falls back to `x264enc` (CPU encoding).

## Notes

- CORS is set to `*` by default. Tighten `allow_origins` in `main.py` for production.
- Tokens are valid for 6 hours (LiveKit default). Adjust in `main.py` via `.with_ttl()` if needed.
- No auth on the `/token` endpoint by default — fine for LAN/private deployments. Add a shared secret check if exposing publicly.
